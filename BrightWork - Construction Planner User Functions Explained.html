Construction Planner User Functions Explained

Construction Planner User Functions Explained

Chart Functions



Main Buttons

Unlock Zoom/Pan (button)

Toggle button to enable/disable zooming and panning of the whole chart. When active, a 4-way cursor will appear over the whole chart and no other functionswill be accessible until the viewport is locked in place. When active, clicking *anywhere* on the viewport will offer the option to lock the viewport in place. When zooming/panning is not enabled, click the button to re-enable it.

Update Resources (button)

Button which displays a userdialog form which does the follow:

  • clears the current 'active' name, if any
  • lists out all names currently on the chart and allows you to remove all data for any or all
  • provides an input box to add names not visible*
  • lists out all current groups beginning with the text 'Gang-'
Click OK to effect the changes, Cancel to exit. Note that clearing the 'active' name will always occur regardless of whichbutton is clicked.

*Only names and groups known to SharePoint can be used - updates applied with unknown names will fail


Clear Finish < Today (button)

This button will offer an 'Ok/Cancel' confirm dialog. Click OK to remove ALL tasks from the chart whose finish dates are before 'today'. This can be used as a quick way to remove unwanted tasks.

Note that 'completed' tasks are neverdisplayed, hence completing tasks and sending the update back to BrightWorkwill ensure these tasks are not displayed in future iterations of the chart.


Shrink (compress) Assignee bars (button)

Click this button and enter the depth of tasks for each assignee bar.


Find Project (button)

Click this button to pop up a form to enter a reference or phrase to find in the chart.


Storage status (information)

Shows Chrome icon if using data from local storage, or XML file icon if using data from saved XML file.


SaveCurrent Tasks (button)

Stores data from current task and bar status, to be recalled on F5 refresh (same browser, same PC only).


Revert to latest XML data (button)

Removes stored data, and replaces currently displayed data with fresh copy from latest XML file.


Add/update Miscellaneous Tasks (button)

This button pops a form with a link to the Projects and Work Tracker site where construction jobs which don't have BrightWork projects can be added. To ensure these tasks appear as bars on the chart, make sure they have the TSC Project Ready Matrix value of 300-Construction selected, and a project reference.

Update Changed Items (button)

This button will offer an 'Ok/Cancel' confirm dialog. Click OK toupdate BrightWork tasks based on changes made in the chart. Changed items are indicated by a green or red border. Only items on projects the user has authority to edit will be updated. See the note in 'Update Resources' about how new assigneesare handled. Care should be taken whenever this button is clicked - changes sent to BrightWorkare permanent and CANNOT BE UNDONE - a notation indicating the changes made and by whom will be added to each changed task.

Get Latest BrightWork data XML (button)

This button will pop a form with a link to click tostart the data exporting process that culminates in the saving of an XML file with the very latest data. It takes a few minutes to complete - when done save the file to the location and with the name specifiedon the form.


Settings


Lock dragged Start to Monday (checkbox)

Checked by default, the chart code forces all startdates to the nearest Monday. If for some reason a date other than Monday is required, uncheck this box before moving the bar.


Lock dragged Finish to Friday (checkbox)


Don't mark (checkbox)

Unchecked by default, check this box if you wish to drag bars around without marking them to update BrightWork (indicated by a Yellow border around the bar). New dates, duration and resource name will still be recorded, but changes to this bar will not be sent to BrightWork.

UseTraining Projects (checkbox)

Checked by default, but remembered upon F5 refresh or returning to the same browser/pc, this checkbox determines whether the chart displays Training Projects or Live Projects.


Descriptions (three-way toggle)

Select one of three options, from longest to shortest. Default is always 'Full'. NB: Find Project will search and findall description text, even if Short or Ref is selected.


Look # years ahead (input)

Enter the number of years of data to include in the chart, default is 2 years.



Task Bar Functions

Complete a Task

Click the circle with checkmark in the top left-hand corner of each bar for a confirmation dialog asking you whether you wish to mark the task as complete. Click OK to mark the task. Note that you must then Click the Update Changed Items button (described above) to send the completion status back to BrightWork.

Remove a Task

Click the circle with the crossmark in the bottom left-hand corner of each bar for a confirmation dialog asking you whether you wish to remove the task from the chart. Note that tasks are only removed from the chart, doing this has no impact on the task in BrightWork.

View Task Information

Click the circle in the top right-hand corner of each bar to view a popup form containinginformation about that task. This form includes a link directly back to the original task in BrightWork. NOte that the colour of this circle will representthe status of Design Unit work according to the keyshown at the top of the chart.

Resize the Task

Click the circle in the bottom right-hand corner to resize the task - a new finish date and duration will be calculated accordingly. Note thaty ou must then click the Update Changed Items button (described above) to send the updated task status to BrightWork.



Resource Functions

IN the context of the Construction Planner 'resource' refers to the task's'Assigned To' name.Movingtasks from one resource sectionto another can update the Assigned To name for the task, however please note the following:

Before moving a task from one resource to another, click on the 'Destination' resource row so it becomes bolder than the other resources - this indicates the resource is now the 'active' resource.

If this is not done before moving a task to a lower section, you may find you can no long click on or dragthe task.

When a user name is active, EVERY task you click will jump to the top of the activeresource's section. To be able to freely drag a taskvertically again, clear the 'active' resource, by clickingthe Update Resources button at the top of the chart (described above, with details below).

Status Highlighting

A summary of bar status highlighting is listed below:

  • Highlighting that indicates thetask will be updated in BrightWork*:

    • Red border - this task has been moved, and is now marked as 'changed'.
    • Green border - this task has been marked as 'completed'.

  • Highlighting for information only (has no additionaleffect when changes are sent back to BrightWork):

    • Yellow border - this task has been moved while the checkbox'Don't mark' has been checked - changes on this task will NOT be sent to BrightWork
    • Red Shadow - (after moving the task) this task's Finish Date exceeds its 'Customer Requireddate'
    • Yellow Shadow - (after moving the task) this task's FinishDate is approaching its 'Customer Required Date'

    • Aqua Shadow - (after moving the task) this task's newStart Date occurs before its original Start Date (confirm with PM/Client)


* Changes will be sent back to BrightWork when the button labeled 'Send all changes back to BrightWork' is clicked.
